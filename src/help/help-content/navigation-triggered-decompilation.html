Navigation-triggered decompilation
When a class file is opened in IntelliJ, one of three things will happen:
  • The class will be decompiled and the source opened in an editor.
  • The class will not be decompiled because naviation-triggered decompilation is disabled, or the class's package has been excluded from decompilation.
  • A dialog will pop up and ask you to confirm the decompilation.
These possibilities are controlled by the decompile-on-navigation setting (see IntelliJad options) and the exclusions setup (see Exclusions).
